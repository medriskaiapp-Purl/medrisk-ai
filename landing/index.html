Risk 2: SpO2 Measurement Error — Missed Hypoxemia Risk ID: RISK_02 Hazard type: Software / Diagnostic Subsystem: Signal processing algorithm (photoplethysmography) Hazardous situation: Device displays falsely normal SpO2 (97%) when patient is hypoxemic (actual SpO2 <90%). Clinician does not intervene. Pre-control risk: P1=3, S1=5 (RPN=15, U — MUST reduce) Risk Control Measures: 2a. Signal Quality Index (Safety by design) Algorithm computes real-time perfusion index (PI) and signal quality index (SQI). When PI <0.3% or SQI <60%, SpO2 value suppressed ("---") and "Low Signal Quality" displayed. No false number shown. 2b. Clinical Accuracy Validation (Safety by design) Validated per ISO 80601-2-61:2017, n≥200 data points, 70-100% SaO2, including Fitzpatrick V-VI. Arms ≤ 3%. 2c. Motion Artifact Rejection (Protective measure) Adaptive filter + accelerometer. Motion >0.5g → slow update + widen CI. Persist >30s → value blanked. Verification: Clinical accuracy: ISO 80601-2-61 desaturation study → Arms ≤ 3.0% SQI blanking: Simulated PI=0.1% → SpO2 blanked within 5s Dark pigmentation bias: Fitzpatrick V-VI vs I-II → Bias ≤ 1.0% Residual Risk: P2=2, S2=5 (RPN=10, A* — benefit-risk justified) IEC Reference: ISO 80601-2-61:2017, IEC 62304
